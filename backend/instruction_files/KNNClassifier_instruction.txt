HYPERPARAMETER_PATTERNS:
•  fixed n_neighbors=5 with uniform weights means all runs use the same neighbourhood size —
  performance variation is entirely data-driven, not tuning-driven
•  Euclidean distance (p=2, minkowski) is optimal for continuous numerical features but degrades
  on encoded categorical features where ordinal distance is not meaningful
•  larger n_neighbors would reduce variance but may lower sensitivity in compact, well-separated clusters
•  smaller n_neighbors would increase sensitivity but amplify noise in low-signal or imbalanced subsets

OBSERVED_MODEL_BEHAVIOR:
•  accuracy improves consistently from 1 to 3 features (mean: 0.655 → 0.876) — KNN benefits strongly
  from additional features that better define neighbourhood structure
•  very high accuracy tier (>=0.95) exclusively from balanced datasets (100%) with near-full numerical subsets
•  imbalance creates the sharpest performance cliff at 3-feature level (mean drops to 0.699) because
  majority-class neighbours dominate voting in imbalanced neighbourhood distributions
•  categorical features encoded as integers introduce artificial ordinal distance, disrupting
  neighbourhood quality and consistently pulling accuracy below numerical-only subsets
•  precision and recall are perfectly balanced (both 0.851) — KNN with uniform weights and balanced data
  produces symmetric class treatment
•  highest ROC AUC (0.895) among all compared models where available, suggesting strong probability
  ranking ability on well-structured numerical data
•  KNN has no training phase — all computation is at inference time, making it memory-intensive
  and slower to predict on large datasets despite fast fitting

MODEL_SELECTION_REASONING:
KNNClassifier should be prioritised for balanced, low-to-moderate dimensional datasets where features
are numerical, scaled, and encoded, and where neighbourhood structure is meaningful (compact, well-separated
clusters). It achieves the highest ROC AUC among compared models and offers symmetric precision-recall
performance. It is particularly effective with 3-feature subsets on datasets with 5–9 or 30+ total features,
where signal density per feature is high and Euclidean distance reliably reflects class proximity.

MODEL_AVOIDANCE_REASONING:
Avoid KNNClassifier for imbalanced datasets — it is completely excluded from the very-high-accuracy tier
under imbalance and suffers an 18.1% accuracy drop at the 3-feature level specifically. Also avoid for
categorical-dominant feature subsets where Euclidean distance on encoded integers does not reflect true
feature similarity. KNNClassifier is not suitable when inference latency or memory efficiency is critical,
as it stores the full training set and computes distances at prediction time. For noisy, sparse, or
high-categorical datasets, CatBoostClassifier is a more robust alternative.
