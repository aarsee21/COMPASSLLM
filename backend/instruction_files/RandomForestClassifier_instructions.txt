# ----------------------------------------------------------
# MODEL SELECTION REASONING
# ----------------------------------------------------------
MODEL_SELECTION_REASONING: >
  RandomForestClassifier should be prioritised for datasets with mixed feature
  types, moderate dimensionality, and non-linear class boundaries.
  Evidence from 6 405 runs shows consistent accuracy above 0.92 on biomedical
  and material-science datasets with well-chosen 2–3 feature subsets. The model
  handles both numerical and categorical inputs natively (with encoding), is
  robust to feature scale, and benefits from ensemble variance reduction via
  bootstrap aggregation. It is particularly effective when a small number of
  features carry strong discriminative signal (e.g. area_mean, concave_points,
  radius features in breast-cancer; Al + Fe in glass identification).

# ----------------------------------------------------------
# MODEL AVOIDANCE REASONING
# ----------------------------------------------------------
MODEL_AVOIDANCE_REASONING: >
  Avoid RandomForestClassifier when interpretability of individual decisions
  is required, as the ensemble of 100 trees is difficult to explain at the
  instance level. Also avoid when inference latency is critical at scale, since
  100-tree evaluation per sample adds overhead. For heavily imbalanced targets,
  use with class_weight: balanced or prefer gradient-boosted alternatives.
  CSV evidence shows accuracy drops to ~0.70 when feature subsets consist
  predominantly of low-signal symmetry or smoothness attributes, indicating
  the model still depends on meaningful feature selection despite its robustness.
