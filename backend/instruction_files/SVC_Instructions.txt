================================================================================
MODEL_SELECTION_REASONING:
================================================================================
SVC (Support Vector Classifier) should be PRIORITIZED for:

✓ Low-dimensional numerical classification problems (2-3 features)
✓ Datasets with compact neighborhood structures and clear boundaries
✓ Properly scaled and preprocessed feature spaces
✓ Biomedical and medical diagnosis classification tasks
✓ Problems requiring high accuracy with limited feature sets
✓ Datasets where overfitting protection is needed
✓ Classification tasks with non-linear decision boundaries
✓ Problems with balanced class distributions
✓ Applications requiring probability estimates
✓ Structured tabular data with numerical features

Key Strengths:
• Exceptional performance (0.91 median accuracy) on well-prepared data
• Robust scaling invariance after feature normalization
• Efficient with limited feature counts
• Strong generalization on compact datasets

================================================================================
MODEL_AVOIDANCE_REASONING:
================================================================================
SVC (Support Vector Classifier) should be AVOIDED for:

✗ High-dimensional sparse datasets (curse of dimensionality)
✗ Datasets with many irrelevant features
✗ Applications with critical inference latency requirements
✗ Very large datasets (computational complexity: O(n²) to O(n³))
✗ Unscaled raw data without proper preprocessing
✗ Problems with significant feature scaling differences
✗ Datasets with severe class imbalance without adjustment
✗ Text classification at scale (use linear classifiers instead)
✗ Streaming/online learning scenarios
✗ Situations requiring feature importance explanations
✗ Highly noisy datasets with unreliable features

Key Limitations:
• Computationally expensive on large datasets (6000+ samples)
• Requires careful feature scaling and preprocessing
• Struggles with irrelevant or noisy features
• Performance degrades rapidly with feature count increase
• Limited interpretability compared to tree-based models

================================================================================
IMPLEMENTATION NOTES:
================================================================================
1. ALWAYS apply feature scaling before SVC training (StandardScaler recommended)
2. Focus on feature selection - quality over quantity (limit to 2-5 best features)
3. Use cross-validation for reliable accuracy estimation
4. Monitor prediction confidence via probability estimates
5. Test multiple kernel types (rbf, linear) for specific datasets
6. Apply class weighting if significant class imbalance exists
7. Consider feature engineering to improve separability
8. Validate on held-out test sets to assess generalization
