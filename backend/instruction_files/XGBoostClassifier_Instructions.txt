================================================================================
MODEL_AVOIDANCE_REASONING:
================================================================================
XGBoostClassifier should be AVOIDED for:

✗ Very small datasets (< 100 samples) prone to overfitting
✗ Extremely high-dimensional datasets (curse of dimensionality)
✗ Text classification without proper feature engineering
✗ Real-time inference with strict latency requirements (< milliseconds)
✗ Scenarios requiring simple, interpretable models (black-box nature)
✗ Cases where model simplicity is mandatory for compliance/regulatory reasons
✗ Problems with extremely sparse features (use linear models instead)
✗ Streaming data requiring online learning capabilities
✗ Datasets with extreme class imbalance without proper handling
✗ Situations with very limited computational resources
✗ Problems solved adequately by simpler linear models (prefer simpler)
✗ Edge deployment on resource-constrained devices

Key Limitations:
• Requires careful feature selection (weak single features perform poorly)
• Computationally more expensive than linear methods during training
• Hyperparameter tuning adds complexity if defaults insufficient
• Model interpretability requires additional analysis (SHAP, feature importance)
• Memory usage scales with ensemble size and feature count
• Training time increases significantly with dataset size
• Prone to overfitting on very small or noisy datasets

1. FOCUS ON FEATURE SELECTION first - quality over quantity
   - Results show 3-feature models vastly outperform 1-2 feature models
   - Irrelevant features can degrade performance

3. USE DOMAIN KNOWLEDGE for feature selection
   - Medical relevance of features significantly impacts results
   - Cholesterol (chol) appears in 7 of top 10 combinations
   - Cardiovascular risk factors complement each other

4. VALIDATE WITH MULTIPLE FEATURE COMBINATIONS
   - Performance varies widely (51% to 100%) based on feature selection
   - Cross-validation essential to find optimal feature sets

5. MONITOR FEATURE IMPORTANCE
   - XGBoost provides feature importance scores
   - Use these to refine feature selection and understand model decisions

6. HANDLE CLASS IMBALANCE if present
   - Use scale_pos_weight parameter to adjust for imbalanced data
   - Not critical for this balanced dataset but important for other scenarios

7. LEVERAGE PROBABILITY ESTIMATES
   - XGBoost provides calibrated probabilities
   - Use for confidence-based decision thresholds

8. CONSIDER EARLY STOPPING
   - Improves generalization and training speed
   - Requires validation set configuration
